# 코드 작업 규칙

코드 작성 및 수정 시 지켜야 할 규칙의 단일 근원.

## 금지 사항

- 사용자가 요청하지 않은 모델 구조 변경 금지
- 요청하지 않는 코딩 금지
- 코딩 전에 프로그램 구조 먼저 제시하고 코딩은 승인 받을 것
- 지시사항을 회피하는 대안 제시 절대 금지

## 코드 수정 규칙

- 기존에 지시한 코드 로직을 임의로 변경하지 않는다.
- 이전에 승인받은 코드 구조를 유지한다.
- 변경이 필요하면 반드시 사유를 설명하고 승인받는다.
- 수정 전 현재 코드 구조를 먼저 파악한다.
- 중복 코드, 중복 함수, 중복 변수가 있는지 조사한다.

## 작업 흐름

1. 사용자 지시 확인
2. 관련 이론·기존 코드 조사
3. 프로그램 구조 제시 → 사용자 승인
4. 구현
5. 2~3 차 점검 (중복·로직 변경·요청 범위)
6. staging (`git add`)
7. 보고 (수행 내역 + 다짐·확인)
